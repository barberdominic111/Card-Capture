# Card Capture

A set-banking, card-stealing capture game for 2–4 players. Built with React + Vite, installable as a PWA.

## Rules summary

- Standard 52-card deck, suits only matter for color/visuals — rank is what counts.
- Capture market cards by **direct rank match** or by **summing two number cards** to match your played card's value.
- **Steal**: if an opponent has 1 or 2 cards of a rank in their capture pile, you can steal them by playing a matching card.
- **Bank**: once you have 3+ of a rank between your capture pile and hand, you may bank them for 1 VP. Banking all 4 grants a bonus turn. You choose when to bank — captured cards sit exposed (and stealable) until then.
- Game ends when the deck is empty and all hands are empty. Most VP wins.

## Local development

```bash
npm install
npm run dev
```

Opens at `http://localhost:5173`.

## Build for production

```bash
npm run build
npm run preview   # sanity check the build locally
```

Output goes to `dist/`.

## Deploying to Vercel

1. Push this repo to GitHub.
2. In Vercel, "Add New Project" → import the repo.
3. Framework preset: **Vite** (auto-detected). Build command `npm run build`, output directory `dist` (these are the Vite defaults, Vercel should pick them up automatically).
4. Deploy. No environment variables needed.

## PWA notes

- Manifest and service worker are generated by `vite-plugin-pwa` at build time — nothing to hand-maintain.
- Icons live in `public/icon-192.png` and `public/icon-512.png`. Replace them with your own art any time; same filenames, square PNGs.
- `autoUpdate` registration means returning visitors get the latest deployed version without manually clearing cache.
- Test the installed-app experience after deploying (not in local dev) — "Add to Home Screen" on mobile Safari/Chrome, or the install icon in desktop Chrome's address bar.

## Modifying the game

Everything — game logic, AI, UI, styling — lives in `src/CardCapture.jsx`. It's a single self-contained component with no external state management; drop in changes and `npm run dev` to see them live.
